Assignment 2

Please Read That Number For Me! (20 points)

Due 1:15 p.m. Monday, September 15

IMPORTANT! This is an individual assignment. You may discuss broad issues of interpretation and understanding and general approaches to a solution. However, conversion to a specific solution or to program code must be your own work. The assignment is expected to be your work, designed and coded by you alone. If you need help, please consult with your instructor or TAs. Specific policies are outlined in Academic Honesty in Computing.

Write a C program spell.c to spell out numbers into their equivalent English text. For example, the number 2014 is read in English as “two thousand fourteen.”

Input: The input consists of one or more positive integers. Each number appears on a separate line and is smaller than a million. You may assume that the input is correct, that is, you do not have check for errors in the input data.

Sample Input

100
1
99999
2014

Output: The output should contain a sorted list of the numbers with their English equivalent phrases.

Sample Output

1: one
100: one hundred
2014: two thousand fourteen
99999: ninety nine thousand nine hundred ninety nine

Programming Notes

  1. Note that scanf() returns the value EOF (defined in stdio.h) if the end of input is reached. Type ‘man scanf‘ for details.
  2. Compile your program with 

    $ gcc -o prog prog.c

    where prog.c is the name of your source code. Your program must conform to the ANSI C Standard, that is, do not compile your program with such flags supporting the C99 standard as -std=c99 or -std=gnu99.

  3. Run your program with 

    $ ./prog < data.in

    where data.in is your input file to the program. The < shell operator redirects the content of the given input file to the standard input. To do this, you need first to create an input data file using a text editor before running your program. Create a text file and enter a positive number. Save the file as data.in. Run your program with the command given above.

  4. Make sure that your program does not prompt for user input.

Documentation

  1. File Comments: Every source file (.h and .c) should have a high-level comment at the top describing the file's contents, and should include purpose of the program, your name and the date. For example: 
    /*
 * Program: hello.c 
 * Purpose: Displays a friendly greeting to students in CPSC 275.
 * Author:  Peter Yoon
 * Date:    9/3/2014
 */
  2. Function Comments: Every function (in both the .h and the .c files) should have a comment describing what function does, what its parameter values are, and what values it returns. For example: 
    /*
 * Function: factorial
 * Purpose:  Computes the factorial of a positive number based on the formula:
 *    n! = 1 * 2 * 3 * ... n
 *
 * Parameters:
 *    n: a positive number
 *
 * Returns: the factorial of n
 */
