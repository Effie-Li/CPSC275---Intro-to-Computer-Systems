IMPORTANT! This is an individual assignment. You may discuss broad issues of interpretation and understanding and general approaches to a solution. However, conversion to a specific solution or to program code must be your own work. The assignment is expected to be your work, designed and coded by you alone. If you need help, please consult with your instructor or TAs. Specific policies are outlined in Academic Honesty in Computing.

  1. CSaPP Homework Problem 3.54. Use Gedit to create a text file (p3-54.txt) which contains your solution to the problem.

  2. CSaPP Homework Problem 3.55. Use Gedit to create a text file (p3-55.txt) which contains your solution to the problem. Make sure to include the assembly code with your annotations.

  3. CSaPP Homework Problem 3.58. Use Gedit to create a text file (p3-58.txt) which contains your solution to the problem.

  4. Numeric Strings. Write a function int2str in IA-32 assembly which, given an integer n, converts n to an equivalent numeric string and returns the address of the string. Note that n can be negative. For example, if n = -1234, the function should return the address of the numeric string “-1234“.

    Write a function int2str in IA-32 assembly which, given an integer n and a pointer to a buffer, converts n to an equivalent numeric string and stores the string in the buffer. Note that n can be negative. For example, if n = -1234, the function should store the numeric string “-1234” in the buffer. To this end, enough space for the buffer must be reserved on the stack and its address must be passed to the int2str function.

    Using this function, write an assembly program (numstr.s) which includes a driver to test your function. Your program should behave as follows:

    Enter an integer: 1234567
The numeric string equivalent to 1234567 is 1,234,567.

    Note that each group of three digits must be separated by a comma. Make sure to annotate the assembly code to show how your program works. Use scanf and printf to format your input and output.
